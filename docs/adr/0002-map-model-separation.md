# ADR-0002: マップロジックをビューから分離した純粋 JS モデルとして実装する

## ステータス

採用

## 背景

マップのポイント割り振りロジックは複雑なルールを持つ。これをコンポーネント内に直接書くとテストが困難になる。

## 決定

`src/model/EthicsMap.js` として、Vue に依存しない純粋な JavaScript クラスとして実装する。

## 理由

- vitest によるユニットテストが書きやすくなる (Vue のマウントが不要)
- ロジックとビューが分離され、それぞれ独立して変更できる
- spec.md の要件「マップの実装はビューのロジックと切り離したモデルとして実現する」に準拠

## 影響

- `src/model/EthicsMap.js` がモデルの実装場所
- Vue コンポーネントはモデルのインスタンスを保持し、`reactive()` や `ref()` でラップして使う
