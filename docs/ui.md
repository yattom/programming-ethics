# UI 設計書

## 画面構成

画面は単一ページ。主要な要素:

1. **倫理マップ** (中央): 同心円状のビジュアル
2. **Pノード** (マップ左側): 縦長の四角形
3. **ノード説明** (マップ右側 or ポップアップ): 選択中ノードの説明テキスト
4. **操作エリア** (マップ下部): ポイント配布開始・追加・リセットボタン

## 同心円マップ

### 円の構造

- 3つの同心円 (内から順にレベル0、レベル1、レベル2)
- 中心円: レベル0 (N0)
- 中間リング: レベル1 (N1-1, N1-2, N1-3)
- 外縁リング: レベル2 (N2-1 〜 N2-6)

### ノード配置 (時計の位置で表現)

| ノード | 配置位置 |
|--------|----------|
| N0     | 中心      |
| N1-1   | 12時      |
| N1-2   | 4時       |
| N1-3   | 8時       |
| N2-1   | 11時      |
| N2-2   | 1時       |
| N2-3   | 3時       |
| N2-4   | 5時       |
| N2-5   | 7時       |
| N2-6   | 9時       |

レベル2のノードは、進化リンクでつながるレベル1ノードの近くに配置する:
- N1-1 (12時) → N2-1 (11時), N2-2 (1時)
- N1-2 (4時) → N2-3 (3時), N2-4 (5時)
- N1-3 (8時) → N2-5 (7時), N2-6 (9時)

## Pノード

- マップの左側に縦長の四角形で表示
- ポイントをトークンで表示 (操作しても変化なし)
- 色は固定 (ポイントに影響されない)

## ポイントのビジュアル表現

### トークン表示

- ノードに割り振ったポイントはトークン (小さなアイコンや丸) を個数で表示
- トークンをクリックしてポイントを減らすことを検討 (要検討)

### 色の濃淡

- ポイントが割り振られると、そのノードと周辺の同心円帯が色づく
- ポイントが多いほど色が濃くなる
- 色は各ノードの帯内に広がり、隣接ノードの色とグラデーションで混じり合う
- 影響範囲の目安:
  - N0: 中央の円全体
  - レベル1ノード: ±2時間分の帯
  - レベル2ノード: ±1時間分の帯

### カラースキーム (要検討)

- ポイントなし: 白
- レベル1: パステル調
- レベル2: ビビッド調
- レベル1とそれに連なるレベル2は近い色合い
- Pノード: 固定色

## ノードのラベルとアイコン

- ノードラベル (N0, N1-1 など) はユーザーに表示しない
- 各ノードの内容を暗示するアイコンを表示してもよい (要検討)

## ノード説明の表示

- ユーザーがノードを選択 (クリックまたはマウスオーバー) すると説明を表示
- 選択中のノードを強調表示 (枠線や輝きなど)
- 説明の表示位置: ポップアップ or 画面上の固定エリア (要検討)

## 操作フロー

### ポイント配布開始

- 初期状態ではポイント配布不可
- 「配布開始」ボタンを押すと Pノードに10ポイントが付与され、配布モードに入る

### ポイント追加 (Pノードへ)

- 「ポイント追加」ボタンで Pノードにポイントを追加できる
- 追加量は要検討

### ポイントのリセット

- 「リセット」ボタンで全ノード (Pノードを含む) のポイントを0に戻す

### URL 共有・保存 (UC004)

- ユーザーが名前を入力できるフィールドを提供
- 「URLをコピー」ボタンで現在の状態を URL に encode してクリップボードにコピー
